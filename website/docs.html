Vibe Pi docs

Everything you need to set up the host agent, pair your device, and read your AI usage at a glance. Vibe Pi runs entirely on your own machine — no cloud account, no telemetry.

Overview

Vibe Pi is a 1.75-inch round AMOLED desk monitor (466×466) that shows how much of your AI coding budget you have left — your rolling 5-hour window and your 7-day window, side by side. It pairs with a small host agent that runs on your computer and reads usage locally.

Local-first by design. The host agent runs on your machine and talks to the device directly over USB or your local network. There is no cloud account to create, nothing is uploaded, and no telemetry is collected. Your usage data never leaves your desk.

  • Device: ESP32-S3, CO5300 display, CST9217 touch, QMI8658 IMU, AXP2101 PMU, Wi-Fi + BLE 5, 16MB flash / 8MB PSRAM.
  • Host agent: Python, runs on macOS, Linux, and Windows.
  • Dashboard + API served locally at http://127.0.0.1:8765.

Getting started

Install the host agent, generate a config, then run it. The agent serves a dashboard and discovers your AI tools automatically.

1. Install the host agent

# clone / enter the host directory, then:
pip install -e '.[web]'

2. Create a config (optional)

Generates a default config file you can edit later.

vibe-pi-host init

3. Run it

vibe-pi-host run
# add --debug for verbose logs

4. Open the dashboard

The agent serves a dashboard and a local API at http://127.0.0.1:8765. From localhost no token is required. The dashboard shows:

  • Live status of detected AI coding tools and your 5-hour / 7-day usage.
  • Paired devices, their connection mode (USB or Wi-Fi), and online state.
  • The pairing screen to confirm a new device, plus the remote-access token for LAN use.
Note
Accessing the dashboard from another machine on your LAN requires the dashboard token — see Pairing.

The macOS app

On macOS you can run the agent as a lightweight menu bar app instead of from the terminal. It lives in the menu bar with no dock-icon clutter and is purely a control surface — it does not re-implement any agent logic, it talks to the running agent over its local HTTP API.

Menu bar

From the menu bar icon you can:

  • Start / Stop the host agent daemon, with a colored status indicator (running / stopped).
  • Open dashboard at http://127.0.0.1:8765.
  • Pair a new device.
  • Copy remote token — copies the LAN access token (~/.config/vibe-pi/dashboard_token) to your clipboard.
  • Launch at login — optional auto-start when you log in.

Native dashboard window

This build adds a built-in native dashboard window via pywebview, so you can view the dashboard without opening a browser. It shows the same content as http://127.0.0.1:8765.

Install & run

# installs the host agent + menu bar deps
pip install -e '.[macapp]'

# run from the menubar directory
python vibe_pi_menubar.py

The menu bar app talks to the agent over its local API at http://127.0.0.1:8765. An agent that auto-starts at login is treated as external — quitting the app won't kill it; it only tears down agents it started itself.

Pairing

Pairing links a device to your host agent so usage data can be sent to it. The device displays a 6-digit code; you confirm it once in the dashboard.

  • Power on the device and connect it (USB cable, or join it to your Wi-Fi during first-time setup).
  • The device shows a 6-digit pairing code.
  • Open the dashboard and confirm that same code. That's it — pairing is one-time.

You can also start pairing from the CLI or the macOS menu bar.

vibe-pi-host pair
Remote / LAN access
Reaching the dashboard from localhost needs no token. Reaching it from another machine on your network requires the dashboard token. Copy it from the macOS menu bar (Copy remote token) or read it from ~/.config/vibe-pi/dashboard_token, then pass it as ?token=… or the X-Vibe-Token header when you open the dashboard over LAN.

Connection modes

Vibe Pi uses dual transport. USB-serial is the primary, zero-config path; Wi-Fi WebSocket is the fallback. The device shows which mode is currently active.

USB-serial (primary)

Plug the device into your computer with a USB-C cable. This path is zero-config: no Wi-Fi credentials, no network discovery. It's the most reliable way to keep the device fed with data.

Wi-Fi WebSocket (fallback)

When USB isn't connected, the device falls back to a WebSocket connection over your local network and finds the host agent at its saved address.

Heads up
mDNS auto-discovery is currently off. Over Wi-Fi the device reaches the host using the saved host IP (and a subnet probe), not automatic discovery. If your host's IP changed, re-provision the device so it learns the new address — see Troubleshooting.

Firmware & updates

The current firmware version is 0.2.3. Updates are delivered as signed OTA images with automatic rollback — if an update fails to boot, the device reverts to the previous known-good firmware on its own.

You can manage firmware images from the CLI:

# publish a firmware file with a version label
vibe-pi-host ota publish firmware.bin 0.2.4

# list published versions
vibe-pi-host ota list

Usage history & CLI

Inspect past usage from the terminal with the history command:

vibe-pi-host history

The full set of subcommands:

  • run — start the host agent and serve the dashboard.
  • init — generate a default config file.
  • devices — list, rename, and unpair devices.
  • pair — begin pairing a new device.
  • ota — publish and list firmware images.
  • cert — manage certificates used for secure connections.
  • history — view recorded usage history.
# a few common examples
vibe-pi-host devices list
vibe-pi-host devices rename <ID> <NAME>
vibe-pi-host devices unpair <ID>

Troubleshooting & FAQ

The device says "wifi failed" +

This usually means the device is trying to reach the host at a stale saved IP address. Because mDNS auto-discovery is currently off, the device relies on the host IP it learned during setup. If your computer's IP changed (new network, DHCP lease, etc.), re-provision the device so it learns the current address — or connect over USB-serial, which needs no IP at all.

It's stuck on "finding host" +

Make sure the host agent is actually running (vibe-pi-host run, or Start from the macOS menu bar) and reachable on the same LAN as the device. The simplest fix is a USB-C cable: USB-serial is the primary, zero-config path and bypasses network discovery entirely.

How do I change screen brightness? +

Brightness is adjustable on the device from its on-screen display settings. The screen also dims and sleeps progressively on its own to stay calm and save power.

How do I factory reset the device? +

Long-press the device button to trigger a reset. The firmware supports staged reset levels, ending with a full factory reset that clears stored settings and pairing so you can start over. You can also unpair a device from the host with vibe-pi-host devices unpair <ID>.

Does any of my data go to the cloud? +

No. Vibe Pi is local-first: the host agent runs on your machine, talks to the device over USB or your LAN, and there is no cloud account and no telemetry. Nothing is uploaded.

Why don't Cursor / Windsurf / Gemini CLI show real numbers? +

Right now only Claude Code and Codex expose real 5-hour and 7-day usage. The others are detected as running so you can see they're active, but accurate usage figures for them are still coming.
