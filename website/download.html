macOS app menu bar

A lightweight menu bar controller that starts and stops the agent, opens the dashboard, and helps you pair — with a native dashboard window so you don't need a browser tab open. It builds on the same vibe-pi-host agent; nothing duplicated.

Download for macOS

Build from source for now. There's no notarized binary yet — follow the steps below to install the menu bar app or build a local .app. A signed download lands in a future release.

Install the menu bar app

  1. 1

    Install the host + app deps

    From the repo, install the agent together with the menu bar extras:

    cd host
pip install -e '.[macapp]'

    This installs the vibe-pi-host agent plus the menu bar dependencies (rumps + pywebview).

  2. 2

    Run the menu bar app

    cd menubar
python vibe_pi_menubar.py

    A dot appears in your menu bar: ○ when the agent is stopped, ● 2 when it's running with 2 AI tools active. Click it to start the agent and open the dashboard.

  3. 3

    Pair and go

    Choose Pair New Device… to confirm the device's 6-digit code, or Open Dashboard Window for a native window with no browser chrome. Turn on Launch at Login to start the agent automatically.

Or build a standalone .app

cd host/menubar
pip install py2app rumps pywebview
python setup.py py2app
open "dist/Vibe Pi.app"

Heads up: the bundle is menu-bar-only (no Dock icon) and expects the vibe-pi-host console script on your PATH. The native dashboard window loads http://127.0.0.1:8765/ and shows a calm fallback page if the agent isn't running.

Host agent any OS

The agent is the core of Vibe Pi — a small Python program that reads your tools' usage locally and talks to the device over USB or Wi-Fi. It runs the same on macOS, Linux and Windows.

1 · Install from source

Clone the repo, then install the agent (a virtualenv is recommended):

cd host
pip install -e .

This puts the vibe-pi-host command on your PATH.

2 · Initialize & run

Generate a config file, then start the agent:

vibe-pi-host init
vibe-pi-host run

It serves a dashboard and API at http://127.0.0.1:8765 — localhost needs no token; LAN access uses one.

Useful commands. vibe-pi-host run · vibe-pi-host init · vibe-pi-host devices · vibe-pi-host pair · vibe-pi-host ota · vibe-pi-host cert · vibe-pi-host history

Firmware v0.2.3

The device ships with firmware v0.2.3. Updates are delivered as signed OTA with automatic rollback — if an update fails to boot cleanly, the device reverts to the previous version on its own, so the monitor on your desk just keeps working.

Already paired?

You don't normally flash by hand. When a new signed build is published, the agent offers it over the air and the device installs it, verifies it, and rolls back automatically if anything's wrong.

Flashing it yourself

Building from source or recovering a board? The flashing and OTA-publish steps live in the docs, including vibe-pi-host ota for publishing your own signed builds.

Pair in three steps

Pairing is a one-time, on-device confirmation. After it, the device reconnects to your agent on its own.

  1. 1

    Start the agent

    Run vibe-pi-host run (or click Start Agent in the macOS menu bar). Power on the device on the same machine or network.

  2. 2

    Read the 6-digit code

    The device shows a 6-digit pairing code on its screen. Keep it visible for the next step.

  3. 3

    Confirm it once

    Enter or confirm the code in the dashboard at http://127.0.0.1:8765 — or pick Pair New Device… in the menu bar. That's it; it reconnects automatically from now on.
